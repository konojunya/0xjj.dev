## How it works

Curl Noise Flow Field は、**WebGPU コンピュートシェーダー**で数万のパーティクルをカールノイズベクトル場に沿って流すビジュアライゼーションです。WebGPU はブラウザから GPU の汎用計算能力に直接アクセスできる次世代 API で、従来の WebGL では困難だった大規模な並列計算をシェーダーコードで記述できます。

### カールノイズとは

カールノイズ（Curl Noise）は、ノイズ関数の **回転（curl）** を取ることで生成されるベクトル場です。通常のノイズをそのまま速度場に使うとパーティクルが一箇所に集まったり発散したりしますが、カールを取ると**発散ゼロ**（div = 0）のベクトル場が得られ、非圧縮流体のようななめらかな流れを実現できます。

```wgsl
// 3D シンプレックスノイズの偏微分からカールベクトルを抽出
let eps = 0.001;
let n  = snoise(vec3(p.x, p.y, t));
let ny = snoise(vec3(p.x, p.y + eps, t));
let nx = snoise(vec3(p.x + eps, p.y, t));
let curl = vec2((ny - n) / eps, -(nx - n) / eps);
```

### コンピュートシェーダーによる並列計算

WebGPU のコンピュートシェーダーは、`@workgroup_size(256)` で 256 スレッドが 1 ワークグループとして同時に実行されます。10 万パーティクルなら約 391 ワークグループがディスパッチされ、GPU 上で一斉に物理演算が走ります。

```wgsl
@compute @workgroup_size(256)
fn main(@builtin(global_invocation_id) gid: vec3u) {
  let i = gid.x;
  if (i >= u.count) { return; }

  var p = src[i];
  // ライフタイムが尽きたらリスポーン
  if (p.life <= 0.0) { respawn(&p, i); }

  // カールノイズでベクトルをサンプリング
  let force = curlNoise(p.px, p.py, u.time) * u.flowSpeed;
  p.vx = p.vx * 0.95 + force.x * u.dt;
  p.vy = p.vy * 0.95 + force.y * u.dt;
  // ...
}
```

### ピンポンバッファ

コンピュートシェーダーは「読み込み元」と「書き込み先」を同じバッファにすると競合が起きます。そこで 2 つのストレージバッファを交互に切り替える**ピンポン方式**を採用しています。フレーム `n` では Buffer A を読み → Buffer B に書き、フレーム `n+1` では逆にします。

### パーティクルの描画

描画はインスタンスレンダリングで、1 パーティクルあたり 6 頂点（2 三角形 = 1 クアッド）を生成し、加算合成（`src: one, dst: one`）で重なりが明るくなるグロウ表現を実現しています。

ライフタイムの残量と速度に応じてアルファが変化し、生まれたてのパーティクルはフェードイン、消える直前はフェードアウトする自然なトランジションを作っています。

### パラメータ

- **ノイズスケール** — ベクトル場の空間的な周波数。大きくすると渦が細かくなる
- **流速** — パーティクルに加わる力の強さ
- **ノイズ変化** — 時間方向のノイズの変化速度。0 にすると静的な場になる
- **パーティクルサイズ** — 各パーティクルのクアッドサイズ
