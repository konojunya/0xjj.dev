## How it works

Air Flow は、Web カメラの映像から **オプティカルフロー**（動きベクトル場）を抽出し、その流れに沿って数万のパーティクルを動かすインタラクティブ可視化です。カメラの前で手を振ると、まるで空気が押し出されるようにパーティクルが流されます。

### オプティカルフローとは

オプティカルフローは、連続する2フレーム間の各ピクセルの動きを推定する画像処理手法です。このデモでは OpenCV.js の **Farneback 法**を Web Worker 上で実行し、メインスレッドをブロックせずにリアルタイム処理を行っています。

```js
cv.calcOpticalFlowFarneback(
  prevGray, gray, flow,
  0.5,   // pyrScale — 画像ピラミッドの縮小率
  3,     // levels — ピラミッドの段数
  15,    // winsize — 平均化ウィンドウのサイズ
  3,     // iterations — 各レベルの反復回数
  5,     // polyN — 多項式展開のサイズ
  1.2,   // polySigma — ガウシアンのσ
  flags
);
```

カメラ映像はフル解像度ではなく **160x120** にダウンサンプルして Worker に渡し、さらに **40x30 グリッド**に集約することで計算コストを大幅に削減しています。

### EMA スムージング

Worker から返ってくるフローデータは毎フレーム微妙に揺らぎます。そこで指数移動平均（EMA）を適用して時間的なジッターを抑えています。

```ts
field.flowX[i] = field.flowX[i] * (1 - EMA_ALPHA) + raw.flowX[i] * EMA_ALPHA;
field.flowY[i] = field.flowY[i] * (1 - EMA_ALPHA) + raw.flowY[i] * EMA_ALPHA;
```

`EMA_ALPHA = 0.3` なので、新しいデータの重みが 30%、前フレームが 70%。なめらかさと反応速度のバランスを取っています。

### パーティクル物理

各パーティクルは正規化座標 `[0, 1]` 上に存在し、毎フレーム以下のステップで更新されます。

1. 現在位置に対応するグリッドセルからバイリニア補間でフローベクトルをサンプリング
2. 速度にダンピング（0.96 倍）を適用しつつフローの力を加算
3. 微弱な重力（`AMBIENT_DRIFT`）を加えて、無風時にもゆっくり落下させる
4. ライフタイムが尽きたか画面外に出たらリスポーン

速度が速いパーティクルほど明るく光り、遅いものは薄くなります。

```ts
const speedBoost = Math.min(1, speed * 200);
const opacity = lifeFade * (0.15 + 0.85 * speedBoost);
```

### WebGL レンダリング

パーティクルは WebGL の `gl.POINTS` として描画され、フラグメントシェーダーで円形にクリッピングしています。ブレンドモードは **加算合成**（`SRC_ALPHA, ONE`）を使い、パーティクルが密集する領域が明るく輝くようにしています。

```glsl
float alpha = smoothstep(0.25, 0.05, d) * v_opacity;
vec3 color = vec3(0.85, 0.9, 1.0); // ソフトブルー
gl_FragColor = vec4(color * alpha, alpha);
```

### パフォーマンス最適化

- フレームスキップ: 毎フレームではなく 3〜4 フレームに 1 回だけ Worker へ送信
- `Transferable` オブジェクト: `ImageData` と `Float32Array` をゼロコピーで Worker と受け渡し
- デバイス適応: デスクトップ 25,000 粒子 / モバイル 5,000 粒子 / 全画面 50,000 粒子
