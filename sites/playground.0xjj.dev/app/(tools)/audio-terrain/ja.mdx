## How it works

Audio Terrain は、マイクの FFT スペクトルをリアルタイムで **3D 地形メッシュ**に変換するビジュアライゼーションです。音楽を流すと、低音域が大きな山脈を、高音域が細かなディテールを生成し、地形が奥に向かってスクロールしていきます。

### FFT から地形へ

Web Audio API の `AnalyserNode` から取得した周波数データを、水平方向（X 軸）に対数スケールでマッピングしています。人間の聴覚は対数的なので、低音域を広く、高音域をコンパクトに配置することで自然な地形になります。

```ts
// 対数スケールで FFT ビンにマッピング
const binF = Math.pow(nx, 2.0) * (binCount * 0.8);
const idx = Math.floor(binF);
const frac = binF - idx;
const val = fftData[idx] * (1 - frac) + fftData[idx + 1] * frac; // 線形補間
```

Treble スライダーで高音域のゲインを調整でき、`freqFactor = 1.0 - nx * (1.0 - trebleK)` で右端（高音）ほど減衰させることも可能です。

### メッシュ構造

地形は `128x128`（モバイルは `64x64`）のグリッドメッシュで、最新のスペクトルが手前の行に書き込まれ、古い行は奥にスクロールしていく**循環バッファ**方式です。全行を毎フレーム再計算する必要はなく、先頭行だけを更新して効率的に動作します。

### ワイヤーフレーム + ブルーム

描画モードは **Solid**（塗りつぶし）と **Wireframe**（ワイヤーフレーム＋ブルーム）の 2 種類。ワイヤーフレームモードでは 2 パスで描画します。

```ts
// Pass 1: ワイヤーフレーム描画
gl.drawElements(gl.LINES, wireCount, indexType, 0);

// Pass 2: 加算合成でブルーム
gl.enable(gl.BLEND);
gl.blendFunc(gl.SRC_ALPHA, gl.ONE);
gl.depthMask(false);
```

ブルームパスでは山頂だけが光る `pow(hn, 3.0)` のべき乗フィルタリングにより、ネオンのような発光エフェクトを実現しています。

### カラーリング

Solid モードでは HSL ベースで高度に応じた色分けを行い、山頂付近に白いグロウを載せています。Wireframe モードでは水平位置で色相を変化させ（左が暖色、右が寒色）、エネルギーに応じてネオンの輝度が上がります。

```glsl
// Wireframe: 周波数位置で色相を決定
float hue = mix(0.0, 0.7, vUv.x) + uTime * 0.02;
// 山頂のネオングロウ
float neon = pow(hn, 2.0) * (1.0 + uEnergy * 2.5);
```

### BPM 同期

ビート検出アルゴリズムが低音域のエネルギー履歴（約 0.75 秒分）の平均と瞬時値を比較してビートを判定。ビート間隔の中央値から BPM を算出し、BPM Sync モードではスクロール速度をテンポに自動同期させます。

### カメラ制御

マウスドラッグでオービット操作（ピッチ・ヨー）ができ、2 秒間操作がないと自動回転に切り替わります。Bass エネルギーでカメラの距離と高さが微妙に揺れ、音楽に合わせて視点が呼吸するように動きます。
